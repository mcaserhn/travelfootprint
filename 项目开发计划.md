# 旅游网站开发计划

## 1. 项目概述

本开发计划基于需求规格说明书，详细规划旅游网站的开发流程，包括项目初始化、核心功能实现、子页面开发、数据管理和测试优化等阶段。

## 2. 开发团队

| 角色 | 职责 | 人员 |
|------|------|------|
| 前端开发 | 负责前端页面和交互功能开发 | 1人 |
| 后端开发 | 负责后端服务和数据管理 | 1人 |
| UI/UX设计 | 负责界面设计和用户体验 | 1人 |
| 测试 | 负责功能测试和性能测试 | 1人 |
| 项目管理 | 负责项目协调和进度跟踪 | 1人 |

## 3. 开发时间线

### 总开发周期：4周

| 阶段 | 时间 | 任务 |
|------|------|------|
| 阶段一：项目初始化 | 第1周 | 搭建项目结构、配置环境、集成基础库 |
| 阶段二：核心功能实现 | 第2周 | 实现微信登录、开发世界地图页面、实现地图交互 |
| 阶段三：子页面开发 | 第3周 | 创建国家子页面、实现照片上传和展示功能 |
| 阶段四：数据管理 | 第3周-第4周 | 集成Firebase、实现数据存储和状态管理 |
| 阶段五：测试和优化 | 第4周 | 功能测试、性能优化、响应式适配、安全性检查 |

## 4. 详细开发计划

### 4.1 阶段一：项目初始化（第1周）

#### 第1天：项目搭建
- 创建Next.js项目
- 配置TypeScript
- 安装Tailwind CSS 3.4

#### 第2天：前端治理规范实施
- 安装必要依赖（Framer Motion、React Hook Form、Zod等）
- 配置Tailwind CSS品牌token（aden-*颜色）
- 创建components/ui目录，实现基线组件（Button、Card、Section、Container）

#### 第3天：地图库集成
- 安装Leaflet.js、react-leaflet和@react-leaflet/core
- 配置地图样式和全局CSS
- 实现基础地图显示

#### 第4天：项目结构设计
- 按照前端治理规范设计目录结构
- 创建基础组件和布局
- 配置App Router路由

#### 第5天：环境配置
- 配置开发环境和环境变量
- 安装Firebase SDK
- 测试基础功能和项目结构

### 4.2 阶段二：核心功能实现（第2周）

#### 第1-2天：微信登录功能
- 集成微信开放平台OAuth 2.0
- 实现WechatLogin组件和登录弹窗（使用Card组件）
- 开发UserContext用户状态管理
- 实现登录状态持久化

#### 第3-4天：世界地图页面
- 实现WorldMap组件和全屏地图布局
- 集成OpenStreetMap瓦片服务
- 添加GeoJSON国家边界数据
- 实现CountryLayer组件
- 实现国家颜色变化逻辑（已上传照片的国家显示aden-orange）

#### 第5天：地图交互功能和UI设计
- 实现鼠标滚轮缩放和地图拖拽
- 开发国家点击事件和路由跳转
- 实现地图悬停效果和提示信息
- 测试地图交互性能和视觉效果

### 4.3 阶段三：子页面开发（第3周）

#### 第1-2天：国家子页面创建
- 实现动态路由/map/[country]
- 设计子页面布局和导航栏（包含国家国旗、名称、返回按钮）
- 开发国家子页面组件（使用Section和Container组件）

#### 第3-4天：照片上传功能和UI设计
- 实现UploadForm组件（使用React Hook Form + Zod）
- 开发文件选择和上传表单（包含拖拽上传区域）
- 集成Firebase Storage上传功能
- 实现上传进度显示（使用aden-orange填充）
- 设计上传按钮和表单样式（使用品牌token）

#### 第5天：照片展示功能和UI设计
- 开发PhotoGallery组件
- 实现照片网格布局（响应式，根据屏幕尺寸调整列数）
- 实现照片预览功能
- 从Firestore获取照片数据
- 设计照片卡片样式（使用Card组件）

### 4.4 阶段四：数据管理（第3周-第4周）

#### 第3周第5天-第4周第2天：Firebase集成
- 创建Firebase项目和配置
- 配置Storage和Firestore服务
- 实现firebase.ts服务文件
- 测试数据连接和基本操作

#### 第4周第3天：用户数据管理
- 实现用户数据存储和同步
- 开发登录状态管理和持久化
- 测试用户数据CRUD操作

#### 第4周第4天：国家状态管理
- 实现国家状态更新逻辑
- 开发地图颜色变化功能
- 测试状态同步和实时更新

### 4.5 阶段五：测试和优化（第4周）

#### 第4周第5天：功能测试和UI/UX测试
- 测试微信登录流程和状态保持
- 测试地图缩放、拖拽和国家点击
- 测试照片上传、展示和删除
- 测试国家状态更新和颜色变化
- 测试UI元素的视觉一致性（品牌token使用）
- 测试交互流程的流畅性

#### 第4周第6天：性能优化和前端治理检查
- 优化地图加载速度（瓦片懒加载、初始缩放限制）
- 优化照片上传速度（压缩、分片上传）

- 检查前端治理规范的实施情况
- 检查代码审查规则的符合性

#### 第4周第7天：响应式适配、安全性检查和最终测试
- 测试不同设备和屏幕尺寸适配
- 检查Firebase安全规则和前端安全
- 测试无障碍设计的符合性
- 最终功能测试和问题修复
- 准备部署和上线

## 5. 技术实现细节

### 5.1 前端实现
- **地图实现**：使用Leaflet.js集成OpenStreetMap瓦片服务，添加GeoJSON国家边界数据实现点击功能，实现WorldMap和CountryLayer组件
- **微信登录**：使用微信开放平台OAuth 2.0授权流程，实现WechatLogin组件和登录弹窗（使用Card组件）
- **照片上传**：使用File API和Firebase Storage，实现UploadForm组件（使用React Hook Form + Zod进行表单处理）和上传进度显示（使用aden-orange填充）
- **照片展示**：使用CSS Grid布局，实现PhotoGallery组件和照片预览功能（响应式网格布局）
- **状态管理**：使用React Context API管理全局状态，包括用户登录状态、已上传照片的国家列表和地图状态
- **路由**：使用Next.js App Router（Server Components 优先），实现主页和动态国家子页面路由
- **样式**：使用Tailwind CSS 3.4实现响应式设计，严格使用品牌token（如bg-aden-orange、text-aden-dark-blue等），禁止使用HEX颜色和inline style
- **动画**：使用Framer Motion实现区块级动画（仅用于页面区块，禁止在UI组件中使用）
- **API**：使用Next.js API Routes实现后端接口
- **组件**：使用统一的基线组件（Button、Card、Section、Container等），禁止页面自造等价样式

### 5.2 UI/UX设计实现
- **视觉设计**：
  - 颜色系统：使用品牌token（aden-orange、aden-dark-blue、aden-digital-blue、aden-dark-grey、aden-mid-grey、aden-light-grey、aden-white）
  - 字体系统：标题使用font-title（Figtree），正文使用font-body（Catamaran），中文使用font-cn（OPPOSans）
  - 布局系统：使用统一的Container组件，避免直接使用固定宽度
- **交互设计**：
  - 微交互：按钮悬停效果（scale-105）、点击效果（scale-95）、表单交互反馈、地图交互反馈
  - 用户流程：清晰的首次访问流程、照片上传流程和浏览流程
- **响应式设计**：
  - 桌面端（1200px+）：4列照片网格，完整功能
  - 平板端（768px-1199px）：3列照片网格，适配屏幕宽度
  - 移动端（360px-767px）：2列照片网格，简化界面
- **无障碍设计**：
  - 语义化HTML结构
  - 适当的颜色对比度
  - 键盘导航支持
  - 屏幕阅读器兼容

### 5.3 前端治理
- **品牌规范**：严格遵循品牌规范-前端工程版，使用品牌token和字体体系
- **组件基线**：使用components/ui目录下的基线组件，确保组件实现一致
- **目录结构**：按照前端治理规范组织项目结构，确保代码组织清晰
- **代码审查**：遵循代码审查规则和清单，确保代码质量和规范符合性

### 5.4 后端实现
- **数据存储**：使用Firebase Storage存储照片，Firestore存储用户数据和国家记录
- **认证**：使用微信开放平台认证，集成Firebase Authentication
- **安全规则**：配置Firebase安全规则，确保数据安全
- **数据模型**：实现用户、照片和国家的数据模型
- **实时同步**：使用Firestore实时数据同步功能

## 7. 风险评估与应对策略

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 微信登录集成问题 | 中 | 高 | 提前申请微信开放平台账号，准备备用登录方案，测试登录流程 |
| 地图性能问题 | 中 | 中 | 优化地图数据加载，使用懒加载技术，限制初始缩放级别，测试不同设备性能 |

| 数据安全问题 | 低 | 高 | 严格配置Firebase安全规则，定期备份数据，加密敏感信息，测试安全规则有效性 |
| 浏览器兼容性问题 | 低 | 中 | 测试主流浏览器，使用polyfill兼容旧浏览器，确保响应式设计适配不同设备 |
