# 前端开发约定（adenexus.com）

> 本约定适用于 **adenexus.com 官网项目**  
> 技术栈：Next.js 14 / React 18 / Tailwind CSS 3 / TypeScript  
> 目标：**品牌一致性、可维护性、可扩展性**
> 技术栈选型依据见《技术栈选型说明.md》

***

## 1. 项目目标（所有开发必须认同）

1.  本项目是 **企业官网**，不是实验场
2.  **品牌一致性 > 个人偏好**
3.  新页面 / 新同事 **不需要额外解释即可上手**
4.  所有样式 **必须可复用、可约束、可审查**

***

## 2. 技术栈边界（禁止擅自变更）

### ✅ 允许使用

*   Next.js 14（App Router）
*   Tailwind CSS 3.4
*   Framer Motion（仅用于区块动画）
*   React Hook Form + Zod（表单）
*   Next API Routes（后端接口）

### ❌ 禁止引入

*   其他 CSS 框架（AntD / MUI / Bootstrap 等）
*   styled-components / emotion
*   行内 style（`style={{}}`，除非极特殊）

***

## 3. 目录与职责约定

```txt
app/            # 页面与路由（Server Components 优先）
components/
  ├── layout/   # Header / Footer / Nav
  ├── sections/ # 页面区块（Hero / Services / CTA）
  ├── ui/       # Button / Card / Input（可复用）
  └── motion/   # Framer Motion 封装
styles/
  └── globals.css
lib/
  ├── zod.ts
  ├── mail.ts
  └── i18n.ts
```

### 约定

*   **页面 = 组合 Section**
*   **Section = 组合 UI 组件**
*   **UI 组件禁止直接访问业务数据**

***

## 4. 样式系统（强制）

### 4.1 颜色使用规范（必须）

✅ **只允许使用 Tailwind tokens**

```tsx
text-aden-orange
bg-aden-dark-blue
border-aden-mid-grey
```

❌ **禁止**

```tsx
text-[#ED4F18]
bg-[#283948]
style={{ color: "#ED4F18" }}
```

> 所有颜色必须来自 `tailwind.config.js`

***

### 4.2 字体使用规范（必须）

*   正文：`font-body`
*   标题：`font-title`
*   中文页面：`font-cn`

❌ 禁止使用：

```tsx
font-sans
font-serif
```

***

### 4.3 文本排版规则（品牌强约束）

❌ 禁止：

```css
text-align: justify;
```

✅ 统一使用自然左对齐（已在 globals.css 固化）

***

## 5. Tailwind 使用约定

### ✅ 推荐写法

```tsx
<section className="py-20">
  <div className="container-content">
    <h2 className="font-title text-3xl">Our Services</h2>
    <p className="mt-4 max-w-prose">
      ...
    </p>
  </div>
</section>
```

### ❌ 不推荐

```tsx
<div className="w-[1180px] mx-auto px-[32px]">
```

> **布局统一使用 `container-content`**

***

## 6. 组件封装规范（非常重要）

> Button / Card / Section 的具体实现与约束
> 以《组件基线说明.md》为唯一依据。


***

## 7. Framer Motion 使用约定（严格）

✅ **只允许用于 Section 级动画**

```tsx
<FadeIn>
  <ServicesSection />
</FadeIn>
```

❌ 禁止

*   Button 动画
*   Input 动画
*   Layout 动画
*   Server Component 中使用 motion

> **动画是点缀，不是主角**

***

## 8. 表单与 API 规范

### 表单

*   所有表单 **必须使用 React Hook Form**
*   所有校验 **必须使用 Zod**
*   禁止前端手写校验逻辑

### API

*   使用 Next.js API Routes
*   前后端共享 Zod schema
*   禁止在页面里直接写邮件 /数据库逻辑

***

## 9. 国际化（i18n）约定

### 路由结构

    /en
    /zh

### 规则

*   结构一致，仅文案变化
*   不允许样式随语言变化
*   文案必须来自统一语言文件

***

## 10. 代码风格与审查原则

### Pull Request 必须检查

*   是否使用了品牌 token
*   是否重复造组件
*   是否违反布局规范
*   是否引入无关依赖

### 评判标准

> ✅ **是否利于半年后继续开发**
> ❌ **是否“我觉得更好看”**

***

## 11. 一句话总结（必须牢记）

> **adenexus.com 不是拼代码能力的地方，是拼“克制”的地方**
