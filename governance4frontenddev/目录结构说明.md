# 目录结构说明

## 项目目录结构

```
app/            # 页面与路由（Server Components 优先）
  ├── about/    # 关于我们页面
  ├── api/      # API 路由
  ├── case-studies/ # 案例研究页面
  ├── compliance/ # 合规页面
  ├── contact/  # 联系页面
  ├── data-sovereignty-statement/ # 数据主权声明页面
  ├── privacy-policy/ # 隐私政策页面
  ├── solutions/ # 解决方案页面
  ├── technology/ # 技术页面
  ├── terms-of-service/ # 服务条款页面
  ├── layout.tsx # 布局组件
  ├── page.server.tsx # 首页（服务端渲染）
  └── page.tsx # 首页（客户端渲染）

components/     # 组件目录
  ├── layout/   # 布局组件（Header / Footer / Nav）
  ├── sections/ # 页面区块（Hero / Services / CTA）
  ├── ui/       # 可复用UI组件（Button / Card / Input等）
  └── motion/   # Framer Motion 封装

lib/            # 工具函数和配置
  ├── i18n.ts  # 国际化配置
  ├── metadata.config.ts # 元数据配置
  └── page-metadata-helper.ts # 页面元数据助手

config/         # 配置文件
  └── site.ts  # 站点配置

locales/        # 国际化语言文件
  ├── ar.json  # 阿拉伯语
  ├── de.json  # 德语
  ├── en.json  # 英语
  └── fr.json  # 法语

public/         # 静态资源
  └── images/  # 图片资源

styles/         # 样式文件
  └── globals.css # 全局样式
```

## 目录职责说明

### app/ 目录
- **页面与路由**：包含所有页面组件和API路由
- **Server Components 优先**：默认使用服务端组件，仅在需要客户端交互时使用 `use client`
- **路由结构**：按照 Next.js App Router 规范组织，支持国际化路由

### components/ 目录
- **layout/**：布局相关组件，如 Header、Footer、导航栏等
- **sections/**：页面区块组件，如 Hero、Services、CTA 等
- **ui/**：可复用的 UI 组件，如 Button、Card、Input 等
- **motion/**：Framer Motion 动画封装组件

### lib/ 目录
- **工具函数**：提供通用的工具函数和辅助方法
- **配置文件**：包含应用配置和设置
- **数据处理**：处理数据获取和转换

### config/ 目录
- **站点配置**：包含站点的基本配置信息

### locales/ 目录
- **国际化语言文件**：存储不同语言的翻译内容

### public/ 目录
- **静态资源**：存储图片、字体等静态文件

### styles/ 目录
- **全局样式**：包含全局样式定义和 Tailwind 配置

## 组件使用规范

1. **页面 = 组合 Section**：页面组件应通过组合 Section 组件来构建
2. **Section = 组合 UI 组件**：Section 组件应通过组合 UI 组件来构建
3. **UI 组件禁止直接访问业务数据**：UI 组件应保持纯展示性，通过 props 接收数据
4. **使用统一组件**：优先使用 components/ui 目录下的统一组件，如 Button、Card 等

## 样式使用规范

1. **使用 Tailwind tokens**：颜色、字体等应使用 Tailwind 配置中定义的 tokens
2. **颜色命名**：使用 `aden-*` 前缀的颜色命名，如 `bg-aden-orange`、`text-aden-dark-blue`
3. **字体命名**：使用 `font-body`、`font-title`、`font-cn` 等统一字体类
4. **布局规范**：使用 `container-content` 类进行布局，避免直接使用固定宽度

## 国际化规范

1. **路由结构**：支持 `/en`、`/fr`、`/ar`、`/de` 等语言前缀
2. **文案管理**：所有文案应从 locales 目录下的语言文件中获取
3. **结构一致**：不同语言版本的页面结构应保持一致，仅文案变化
