# Code Review CheckList（adenexus.com）

> 用于 Pull Request 审查，确保代码符合前端开发约定
> 本清单为《代码审查规则.md》的操作化版本，
> 不新增、不解释、不修改任何规则。
> 如有冲突，以《代码审查规则.md》为准。

---

## ✅ 代码审查清单

### 1. 技术栈检查

- [ ] 是否只使用了允许的技术栈（Next.js 14 / Tailwind CSS 3 / React 18 / TypeScript）
- [ ] 是否引入了禁止的库（AntD / MUI / styled-components 等）
- [ ] 是否使用了行内 `style={{}}`（除非极特殊场景）

---

### 2. 目录结构检查

- [ ] 页面文件是否放在 `app/` 目录下
- [ ] 布局组件是否放在 `components/layout/` 目录下
- [ ] 通用 Section 组件是否放在 `components/sections/` 目录下
- [ ] UI 组件是否放在 `components/ui/` 目录下
- [ ] Motion 组件是否放在 `components/motion/` 目录下

---

### 3. 样式系统检查

#### 颜色使用
- [ ] 是否只使用了 `aden-*` 颜色 token
- [ ] 是否有硬编码颜色值（如 `#ED4F18`、`#283948`）
- [ ] 是否使用了正确的颜色命名：
  - `bg-aden-dark-blue`（深蓝背景）
  - `text-aden-orange`（橙色文字）
  - `text-aden-dark-grey`（深灰文字）
  - `bg-aden-light-gray`（浅灰背景）

#### 字体使用
- [ ] 标题是否使用了 `font-title`
- [ ] 正文是否使用了 `font-body`
- [ ] 中文页面是否使用了 `font-cn`
- [ ] 是否避免了 `font-sans`、`font-serif`

#### 布局使用
- [ ] 是否使用了 `container-content` 作为统一布局类
- [ ] 是否避免了硬编码宽度（如 `w-[1180px]`）
- [ ] 是否避免了硬编码 padding（如 `px-[32px]`）

---

### 4. 组件使用检查

#### Button 组件
- [ ] 是否使用了 `Button` 组件而不是原生 `<button>`
- [ ] 按钮变体是否正确（`primary` / `dark` / `outline`）
- [ ] 是否避免了直接写按钮样式（如 `bg-aden-orange px-6 py-3`）

#### Card 组件
- [ ] 卡片是否使用了 `Card`、`CardHeader`、`CardBody`、`CardFooter` 组件
- [ ] 是否避免了临时造卡片样式（如 `bg-white rounded-lg shadow-md`）

#### Section 组件
- [ ] 是否优先使用封装的 Section 组件（`HeroSection`、`CTASection`、`ContentSection`）
- [ ] 页面是否是 Section 组件的组合，而不是一锅粥

---

### 5. Framer Motion 使用检查

- [ ] 动画是否只用于 Section 级，而不是 Button/Input 等小组件
- [ ] 是否使用了封装的 Motion 组件（`FadeIn`、`SlideIn`、`ScaleIn`、`StaggerGrid`）
- [ ] 是否在 Server Component 中避免了 motion
- [ ] 动画参数是否合理：
  - `duration` 是否在 0.3-0.8 秒之间
  - `viewport={{ once: true }}` 是否设置（避免重复动画）
  - `threshold` 是否合理（默认 0.1）

---

### 6. 国际化检查

- [ ] 文案是否来自 `translations` 对象
- [ ] 是否避免了硬编码文案（特别是多语言文案）
- [ ] 路由是否正确处理了语言前缀（`/en/...`、`/zh/...`）
- [ ] 样式是否独立于语言（不随语言变化）

---

### 7. 表单与 API 检查

#### 表单
- [ ] 是否使用了 React Hook Form
- [ ] 校验是否使用了 Zod
- [ ] 是否避免了手写校验逻辑

#### API
- [ ] 是否使用了 Next.js API Routes
- [ ] 是否避免了在页面中直接写邮件/数据库逻辑
- [ ] 前后端是否共享 Zod schema

---

### 8. 代码风格检查

- [ ] 代码是否简洁、可读
- [ ] 是否有不必要的注释（代码应该自解释）
- [ ] 是否有 dead code（未使用的代码）
- [ ] 是否有 console.log（应该删除）

---

### 9. 性能检查

- [ ] 图片是否使用了优化的格式和尺寸
- [ ] 是否避免了不必要的大图
- [ ] 是否使用了 Next.js 的优化组件（如 `Image`、`Link`）

---

### 10. 可维护性检查

- [ ] 代码是否利于半年后继续开发
- [ ] 新同事是否能无需额外解释即可上手
- [ ] 样式是否可复用、可约束
- [ ] 组件职责是否清晰

---

## 🚩 常见错误示例

### ❌ 错误示例

```tsx
// 硬编码颜色
<div className="bg-[#283948] text-[#ED4F18]">

// 硬编码布局
<div className="w-[1180px] mx-auto px-[32px]">

// 临时造按钮
<button className="bg-aden-orange px-6 py-3 text-white rounded">

// 临时造卡片
<div className="bg-white rounded-lg shadow-md p-6">

// 行内样式
<div style={{ color: '#ED4F18' }}>

// 直接使用 font-sans
<p className="font-sans">
```

### ✅ 正确示例

```tsx
// 使用颜色 token
<div className="bg-aden-dark-blue text-aden-orange">

// 使用统一布局
<div className="container-content">

// 使用 Button 组件
<Button variant="primary">Contact us</Button>

// 使用 Card 组件
<Card>
  <CardHeader>...</CardHeader>
  <CardBody>...</CardBody>
</Card>

// 使用封装的 Motion 组件
<FadeIn>
  <ContentSection title="...">...</ContentSection>
</FadeIn>

// 使用正确的字体
<p className="font-body">
<h1 className="font-title">
```

## 📋 审查流程

1. **自查**：提交前对照本清单自查
2. **机器审查**：运行 lint 和 typecheck
3. **人工审查**：至少一人审查
4. **测试验证**：在开发环境测试功能
5. **合并**：审查通过后合并

---

## 🎯 评判标准

> ✅ **是否利于半年后继续开发**
> 
> ❌ **是否"我觉得更好看"**

> **adenexus.com 不是拼代码能力的地方，是拼"克制"的地方**
