## 2. 品牌核心原则（工程视角）

### 2.1 品牌气质关键词（工程抽象）

*   **Professional / Engineering-driven**
*   **Human-centric**
*   **Clean & Restrained**
*   **No visual noise**

> 工程含义：  
> ✅ 克制  
> ✅ 少变化  
> ✅ 不追求“炫”

***

### 2.2 工程层的总体约束

*   **品牌一致性 > 页面个性**
*   **统一优先于灵活**
*   **工程可维护性 > 视觉微调**

***

## 3. 品牌颜色（Brand Colors）

### 3.1 官方颜色定义（唯一来源）

| 语义           | HEX       | Tailwind Token      |
| ------------ | --------- | ------------------- |
| Brand Orange | `#ED4F18` | `aden-orange`       |
| Dark Blue    | `#283948` | `aden-dark-blue`    |
| Digital Blue | `#1EE6FF` | `aden-digital-blue` |
| Dark Grey    | `#333333` | `aden-dark-grey`    |
| Mid Grey     | `#D1D1D1` | `aden-mid-grey`     |
| Light Grey   | `#F8F8F8` | `aden-light-grey`   |
| White        | `#FFFFFF` | `aden-white`        |

***

### 3.2 工程使用规则（强制）

✅ **只允许使用 Tailwind Token**

```tsx
text-aden-orange
bg-aden-dark-blue
border-aden-mid-grey
```

❌ **禁止**

```tsx
text-[#ED4F18]
bg-[#283948]
style={{ color: "#ED4F18" }}
```

> **任何 HEX / RGB / inline style 均视为违规**

***

## 4. 字体体系（Typography）

### 4.1 官方字体角色

| 用途 | 字体        | Tailwind Token |
| -- | --------- | -------------- |
| 标题 | Figtree   | `font-title`   |
| 正文 | Catamaran | `font-body`    |
| 中文 | OPPOSans  | `font-cn`      |
| 兜底 | Arial     | 自动             |

***

### 4.2 工程使用规则

✅ 合法：

```tsx
<h1 className="font-title" />
<p className="font-body" />
```

❌ 不允许：

```tsx
font-sans
font-serif
```

***

### 4.3 排版禁令（品牌强约束）

❌ **禁止正文两端对齐**

```css
text-align: justify;
```

✅ 全站正文统一左对齐  
（已在 `globals.css` 中固化）

***

## 5. 布局与留白（Layout & Spacing）

### 5.1 核心原则

*   页面宽度必须统一
*   留白优先使用垂直 spacing
*   避免精确像素微调

***

### 5.2 工程规范

✅ **必须使用统一容器**

```tsx
<Container>
  ...
</Container>
```

❌ 禁止：

```tsx
<div className="w-[1180px] mx-auto px-[32px]">
```

***

## 6. 组件层级与品牌映射

### 6.1 品牌 → 组件映射关系

| 品牌概念    | 工程实现                |
| ------- | ------------------- |
| 强调 / 行动 | Button              |
| 信息模块    | Card                |
| 页面结构    | Section + Container |

***

### 6.2 组件基线（强制）

以下组件被定义为**品牌基线组件**：

*   `Button`
*   `Card`
*   `Section / Container`

规则：

1.  页面必须优先使用基线组件
2.  不允许页面自造等价样式
3.  不允许复制、派生组件
4.  如需扩展，必须在原组件中完成

***

## 7. 动效（Motion）品牌边界

### 7.1 品牌原则

*   动效用于**节奏引导**
*   不用于装饰
*   不喧宾夺主

***

### 7.2 工程规则

✅ 允许：

*   Section 级 FadeIn
*   统一封装的 motion 组件

❌ 禁止：

*   Button / Input 动画
*   UI 组件内使用 motion
*   Server Component 中使用 motion

***

## 8. 允许变化 vs 禁止变化

### 8.1 允许变化（在规则内）

*   Section 顺序
*   内容组合
*   文案长度
*   模块是否出现

***

### 8.2 禁止变化（红线）

*   颜色体系
*   字体体系
*   按钮样式
*   页面宽度规则
*   基线组件结构
