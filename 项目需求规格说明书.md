# 旅游网站需求规格说明书

## 1. 项目概述

### 1.1 项目背景
随着全球旅游的普及，用户希望有一个直观的平台来记录和展示自己的旅游经历。本项目旨在构建一个基于世界地图的旅游记录网站，用户可以通过微信登录后，在不同国家上传旅游照片，并通过地图可视化展示自己的旅行足迹。

### 1.2 项目目标
- 提供微信登录功能，简化用户注册登录流程
- 实现交互式世界地图，支持缩放和国家点击
- 允许用户在国家子页面上传旅游照片
- 通过颜色变化直观显示用户已上传照片的国家

## 2. 功能需求

### 2.1 用户认证
- **F1.1 微信登录**：网站加载时自动弹出微信登录弹窗
- **F1.2 登录状态管理**：保持用户登录状态，支持自动登录

### 2.2 地图功能
- **F2.1 世界地图展示**：主页显示完整世界地图
- **F2.2 地图交互**：支持鼠标滚轮缩放、拖拽移动
- **F2.3 国家点击**：点击地图上的国家进入对应子页面
- **F2.4 国家状态显示**：未上传照片的国家显示白色，已上传照片的国家显示其他颜色

### 2.3 国家子页面
- **F3.1 照片上传**：支持用户上传在该国家的旅游照片
- **F3.2 照片展示**：展示用户在该国家上传的所有照片
- **F3.3 返回主页**：提供返回世界地图主页的功能

### 2.4 数据管理
- **F4.1 照片存储**：存储用户上传的照片
- **F4.2 用户足迹记录**：记录用户已上传照片的国家
- **F4.3 数据持久化**：确保数据在用户刷新页面后仍然保留

## 3. 非功能需求

### 3.1 性能需求
- **NF1.1 地图加载速度**：世界地图页面加载时间不超过3秒
- **NF1.2 照片上传速度**：单张照片上传时间不超过10秒（基于5MB照片）
- **NF1.3 页面响应时间**：页面操作响应时间不超过1秒

### 3.2 用户体验
- **NF2.1 界面美观**：采用现代化设计风格，视觉效果良好
- **NF2.2 交互流畅**：地图缩放和拖拽操作流畅无卡顿
- **NF2.3 响应式设计**：适配不同屏幕尺寸的设备

### 3.3 安全性
- **NF3.1 数据安全**：用户照片和个人信息安全存储
- **NF3.2 认证安全**：微信登录流程安全可靠

## 4. 技术选型

### 4.1 前端技术
- **框架**：Next.js 14（App Router，Server Components 优先）
- **语言**：TypeScript
- **地图库**：Leaflet.js（支持全球地图覆盖，通过集成OpenStreetMap、Mapbox等第三方瓦片服务实现全球国家和地区的显示，支持多级缩放、交互操作和自定义图层）
- **样式**：Tailwind CSS 3.4（使用品牌token，如bg-aden-orange、text-aden-dark-blue等，禁止使用HEX颜色和inline style）
- **状态管理**：React Context API
- **表单**：React Hook Form + Zod
- **动画**：Framer Motion（仅用于区块级动画，禁止在UI组件中使用）
- **图片上传**：使用浏览器原生File API
- **API**：Next.js API Routes
- **组件库**：使用统一的基线组件（Button、Card、Section、Container等）

### 4.2 后端技术
- **数据存储**：Firebase Storage（存储照片）
- **数据库**：Firebase Firestore（存储用户数据和国家记录）
- **认证**：微信开放平台认证

### 4.3 品牌规范
- **颜色体系**：使用品牌token（aden-orange、aden-dark-blue、aden-digital-blue、aden-dark-grey、aden-mid-grey、aden-light-grey、aden-white）
- **字体体系**：标题使用font-title（Figtree），正文使用font-body（Catamaran），中文使用font-cn（OPPOSans）
- **布局规范**：使用统一的Container组件，避免直接使用固定宽度
- **组件规范**：使用基线组件，禁止页面自造等价样式

## 5. 页面设计

### 5.1 主页（世界地图）
- **布局**：全屏世界地图
- **元素**：
  - 顶部导航栏（fixed定位，半透明背景，显示用户信息、登录状态）
  - 世界地图（支持缩放、拖拽，国家边界清晰）
  - 国家区域（可点击，已上传照片的国家显示不同颜色）
  - 登录弹窗（首次访问时自动弹出，使用Card组件，居中显示）
- **视觉设计**：
  - 地图风格：OpenStreetMap标准风格，浅色背景
  - 国家边界：1px实线，颜色为aden-mid-grey
  - 已上传照片国家：填充色为aden-orange，透明度60%
  - 悬停效果：国家边界加粗至2px，颜色变为aden-dark-blue

### 5.2 国家子页面
- **布局**：上方导航栏 + 照片上传区域 + 照片展示区域
- **元素**：
  - 顶部导航栏（包含国家国旗、名称、返回按钮）
  - 照片上传区域（居中显示，包含拖拽上传区域和文件选择按钮）
  - 照片展示区域（响应式网格布局，根据屏幕尺寸调整列数）
- **视觉设计**：
  - 上传按钮：使用品牌橙色primary按钮
  - 照片卡片：使用Card组件，包含照片、上传时间和删除按钮
  - 返回按钮：使用outline按钮，左侧箭头图标

### 5.3 UI/UX设计要求
- **视觉设计系统**：
  - 颜色系统：使用品牌token（aden-orange、aden-dark-blue、aden-digital-blue、aden-dark-grey、aden-mid-grey、aden-light-grey、aden-white）
  - 字体系统：标题使用font-title（Figtree），正文使用font-body（Catamaran），中文使用font-cn（OPPOSans）
  - 布局系统：使用统一的Container组件，避免直接使用固定宽度
- **交互设计**：
  - 微交互：按钮悬停效果、表单交互反馈、地图交互反馈、上传进度显示
  - 用户流程：清晰的首次访问流程、照片上传流程和浏览流程
- **响应式设计**：
  - 桌面端（1200px+）：4列照片网格，完整功能
  - 平板端（768px-1199px）：3列照片网格，适配屏幕宽度
  - 移动端（360px-767px）：2列照片网格，简化界面
- **无障碍设计**：
  - 语义化HTML结构
  - 适当的颜色对比度
  - 键盘导航支持
  - 屏幕阅读器兼容

## 6. 数据结构

```typescript
interface User {
```

```typescript
interface Photo {
  id: string;            // 照片唯一标识

  country: string;       // 国家名称
  url: string;           // 照片存储URL
  description?: string;  // 照片描述（可选）
  createdAt: timestamp;  // 上传时间
}
```

```typescript
interface Country {
  code: string;          // 国家代码
  name: string;          // 国家名称
  hasPhotos: boolean;    // 是否有用户照片

```

## 9. 风险评估

### 9.1 潜在风险
- **微信登录集成**：可能遇到微信开放平台审核和配置问题
- **地图性能**：大量国家数据可能影响地图加载速度
- **照片存储**：大量照片上传可能导致存储成本增加

### 9.2 风险缓解措施
- 提前申请微信开放平台账号并完成审核
- 优化地图数据加载，使用懒加载技术
- 实现照片压缩和合理的存储策略
